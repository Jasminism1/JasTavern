# SillyTavern Custom UI Framework (Vue)

这个项目是一个基于 Vue 3 + TypeScript + Vite + Pinia 开发的自定义前端应用，专门用于接入 SillyTavern（酒馆）。它的主要目的是为了替换原本的酒馆前端界面，为复杂或大型的角色卡提供一种深度沉浸、拥有定制交互体验的专属游戏 UI。

## 期望实现的效果

- **无缝接入**：玩家在 SillyTavern 导入特定的角色卡后，在原生的酒馆信息界面上会出现一个由酒馆插件注入的“进入游戏”按钮。
- **沉浸式体验**：点击按钮后，原生的酒馆界面将被覆盖，玩家进入这个精美的全屏 Vue 应用中进行游戏互动。
- **复杂的界面布局**：应用主要采用三栏经典布局（左侧立绘/Live2D展示，中间对话流及输入区域，右侧角色状态及属性），并辅以顶部导航及各类拓展面板（如角色详情、事件日志、地图、设置等）。
- **实时通信与状态同步**：该前端框架需要与 SillyTavern 环境保持数据同步，获取角色的回复、状态变化及其他游戏内事件，并更新至 Vue 的状态管理中。

## 开发规范 (Norms & Specifications)

1. **技术栈要求**：
   - 核心框架：Vue 3 (Composition API, `<script setup>` 语法)。
   - 强类型：全面使用 TypeScript 进行类型声明，确保组件间通信和状态管理的安全与可维护性。
   - 状态管理：使用 Pinia。状态按功能模块化划分（如 `app.ts` 处理全局 UI 状态，`background.ts` 处理背景，`settings.ts` 处理设置等）。
   - 构建工具：Vite。
2. **代码风格与组件规范**：
   - 组件拆分：根据功能拆分独立的组件，放置于 `src/components/` 目录下。保持组件职责单一。
   - 样式规范：优先使用 Scoped 样式避免污染全局。界面颜色以深色调（如 `#0a0a12`, `#1a2535`）为主，匹配沉浸式游戏 UI 的氛围。
   - 响应式与交互：利用 Vue 的 `<Transition>` 增加界面切换、元素出现的过渡动画，提供细腻的微交互效果（如悬浮高亮、点击反馈）。
3. **架构与解耦**：
   - **通信桥接 (Bridge 层)**：由于项目最终要运行在 SillyTavern 环境中，一切与酒馆原生上下文交互的代码必须通过封装好的 Bridge 层（例如建立 `src/bridge/` 目录）实现，避免在 UI 组件中直接耦合酒馆的全局变量或 DOM 操作。
   - **资产管理**：图像、音效等静态资产优先使用酒馆侧暴露的静态资源服务，或者存放在 `src/assets/` 中以确保打包无误。

## 开发流程 (Development Workflow)

1. **本地环境准备与调试**：
   - 确保安装 Node.js。
   - 运行 `npm install` 安装所有依赖。
   - 运行 `npm run dev` 启动本地开发服务器进行独立调试。此时可通过 mock（模拟）数据来模拟与酒馆的通信，确保前端逻辑跑通。
2. **核心业务逻辑与界面开发**：
   - 继续开发和完善 `DialoguePanel`, `StatusPanel`, `PortraitLayer` 等核心面板组件。
   - 细化交互和状态流转，如对话流的自动滚动、角色属性值的动态更新。
3. **实现通信层 (Bridge)**：
   - 编写与 SillyTavern 的双向通信接口（可能是通过酒馆插件提供的事件监听机制，或是基于扩展 API 的调用）。
   - 将 Pinia 的 store 与 Bridge 层连接，实现状态在收到酒馆消息时的自动更新。
4. **样式与视觉打磨**：
   - 调整排版、色彩方案和特效，使之符合“精美、高级感”的视觉要求（可加入背景模糊、玻璃态组件、平滑的展开折叠动画等）。
5. **打包与集成测试**：
   - 运行 `npm run build` 生成构建产物。
   - 将打包后的页面及脚本作为酒馆扩展（Extension）的一部分加载，或通过 iframe / Shadow DOM 嵌入 SillyTavern 中进行实际运行测试。
   - 验证游戏内上下文获取、消息发送接收、立绘更新等核心链路。
6. **性能优化**：
   - 处理长对话记录的渲染性能（必要时引入虚拟列表机制）。
   - 检查组件销毁时的事件解绑，防止内存泄漏。
